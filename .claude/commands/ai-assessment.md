# /ai_assessment [client_name] [criterion]

Generate a structured Accredited Investor (AI) eligibility assessment memo for internal RM use.

## Purpose

This command produces a formal, auditable AI eligibility assessment for a named client under a single RM-selected criterion. The output is a draft compliance memo for checker review — not a final determination of AI status.

All assessment logic is deterministic. Claude renders the memo from a structured decision object; Claude cannot alter the pass/fail result.

## Criterion Selection

The RM must specify which criterion to assess under. Three are available:

| Option | Criterion | SGD Threshold |
|--------|-----------|---------------|
| 1 | Annual Income | ≥ SGD 300,000 |
| 2 | Net Personal Assets | > SGD 2,000,000 |
| 3 | Net Financial Assets | > SGD 1,000,000 |

**Do not auto-switch.** If the selected criterion fails, the result is fail for that criterion. Do not evaluate or suggest an alternative criterion unless the RM explicitly requests it.

## Data Source

Assessment data is loaded from the `AI_Assessment` tab in Google Sheets for the named client. If no record exists, instruct the RM to run the bootstrap script and populate the client's data.

## Locked Business Rules

These rules are enforced by the deterministic engine and must not be overridden in the memo:

1. **Single-criterion only** — assess exactly the criterion selected by the RM
2. **No auto-switching** — if criterion fails, return fail for that criterion only
3. **Source control gate** — external manually-entered values without `evidence_type` + valid `evidence_date` are excluded from recognised amount and cannot support a pass
4. **Internal bank-held values** (`source_is_internal = True`) may support pass without separate evidence
5. **Primary residence cap** — net equity (after secured loan, pro-rated for ownership share) capped at SGD 1,000,000 in NPA computation
6. **CPFIS only** — CPFIS-eligible investments are included; raw CPF balances (OA/SA/MA) are explicitly excluded
7. **Raw CPF excluded** — never include CPF balances other than CPFIS in NFA
8. **DPT/stablecoin excluded** — digital payment tokens and stablecoins are not schema-supported; do not reference them
9. **Borderline pass (within 10% above threshold)** — forced to `manual_review`; confidence capped at Medium
10. **Joint account** — flag only; no separate qualification path for jointly-held assets in this version
11. **Opt-in/opt-out** — out of scope; do not generate, reference, or request opt-in decisions
12. **Draft only** — this is a draft for checker review; checker confirms or rejects; never write "approved as Accredited Investor"

## Evidence Recency Limits

| Evidence Type | Maximum Age |
|---------------|-------------|
| income_statement | 60 days |
| employer_letter | 60 days |
| external_investment_statement | 60 days |
| letter_of_financial_standing | 90 days |
| property_valuation | 365 days |
| NOA | Supplementary only — not sufficient as primary evidence |

Stale evidence forces `manual_review`.

## Output Format

The output has two parts, in order:

### Part 1 — Summary Card (deterministic)

Rendered by Python, not Claude. Do not modify:

```
*AI Assessment — {client name}*

*Criterion:* [criterion label]
*Recognised Amount:* SGD X,XXX,XXX
*Threshold:* SGD X,XXX,XXX
*Outcome:* ✅/❌/⚠️/⏳ [status]
*Confidence:* HIGH / MEDIUM / LOW

*Missing Items:*
  [list or "None"]

*Flags:*
  [list or "None"]

*Checker Recommendation:*
  [recommendation text]

_Checker status: pending_review | For internal RM use only._
```

### Part 2 — Compliance Memo (Claude-drafted)

Use this exact section order. Do not reorder, add, or remove sections:

```
*Accredited Investor Assessment — {client name}*
_Draft memo for checker review_

*Eligibility Basis*
[criterion and threshold — one sentence]

*Recognised Amount*
[exact SGD figure and whether it meets threshold]

*Evidence and Source*
[source type, date, internal vs external]

*Validation Notes*
[checks performed; name all flags explicitly or confirm clean]

*Missing Information*
[each missing field listed, or "No missing required fields"]

*Assessment Outcome*
✅/❌/⚠️/⏳ [status display]
Confidence: HIGH / MEDIUM / LOW
[1–2 sentences reasoning — no invented logic or values]

*Manual Review*
[list manual_review_reasons if any, or "No manual review required"]

*Checker Recommendation*
[exact text from AIDecisionResult.checker_recommendation]

*RM Declaration*
This memo was generated by Aureus RM Copilot based on data provided by the relationship manager. The RM is responsible for verifying the accuracy of all input data and for obtaining the client's signed Accredited Investor declaration form prior to any reclassification.

*Risk Disclosure*
Accredited Investors are subject to reduced regulatory safeguards under the Securities and Futures Act (Cap. 289). This document does not constitute a determination of eligibility. Final sign-off requires review and approval by an authorised checker.

_Draft for internal RM use only. Not for client distribution. Checker status: pending_review._
```

## Language Rules

**Never write:**
- "approved as Accredited Investor"
- "AI-approved"
- "final approval"
- "customer qualifies overall"
- "it appears", "it seems", "broadly", "generally"

**Always write:**
- "eligible under selected criterion"
- "not eligible under selected criterion"
- "pending information"
- "manual review required"
- "draft memo for checker review"

## Missing Data Handling

| Missing Data | Required Response |
|-------------|------------------|
| No AI_Assessment record | "Client record not found in AI_Assessment tab. RM to populate data before running assessment." |
| No criterion selected | Ask RM to specify 1, 2, or 3 before proceeding |
| Critical field absent | assessment_status = pending_info; state what is missing |
| Evidence stale or absent | Flag in Validation Notes; force manual_review |

## Compliance

Read `.claude/rules/compliance.md` before generating any output. The standard compliance disclaimer must appear at the bottom of every assessment output.
